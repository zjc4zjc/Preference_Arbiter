 # 🗂 LOG：按周记录实验结果

> **约定：**  
> - 本周的周一到下周的周一对应一个 Markdown 文件，命名为：`YYYY.MM.DD~YYYY.MM.DD.md`，例如：`2026.03.09~2026.03.16.md`。  
> - 一周内的所有实验，全部记在该周的文件里，由 Agent 自动或半自动补充。

---

## 1. 文件命名规则

- 采用年月日编号：  
  - 年份：`YYYY`（例如 `2026`）  
  - 月份：`MM`（例如 `03` 表示 3 月）  
  - 日期：`DD` (例如 `09` 表示 9 号)

---

## 2. 每周日志模板（建议结构）

每个 `YYYY.MM.DD~YYYY.MM.DD.md` 推荐包含以下几个部分：

1. **本周概览（Weekly Overview）**
2. **按实验 ID 记录详情（Experiments）**
3. **本周总结与下周建议（Summary & Next Vibe）**

---

## 3. 示例：

可以复制该文件作为之后每周的起始模板，再按需调整。  
Agent 在运行新实验时，应自动在对应文件中追加一条实验记录。
